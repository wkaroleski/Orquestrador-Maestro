# Best Practices

> Development best practices including clean code, SOLID principles, and coding standards

## When to Use

- Code reviews
- New projects
- Team onboarding
- Quality improvement

## Principles

### Clean Code
- Meaningful names
- Small functions
- Single responsibility
- Comments when needed

### SOLID
- **S**ingle Responsibility
- **O**pen/Closed
- **L**iskov Substitution
- **I**nterface Segregation
- **D**ependency Inversion

### Naming
- Variables: camelCase
- Constants: SCREAMING_SNAKE
- Components: PascalCase
- Files: kebab-case
