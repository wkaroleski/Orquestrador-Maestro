# Deep Wiki

Map the repository from evidence. Cite source files, distinguish facts from inference, and preserve existing documentation conventions. Generate an index, architecture overview, setup and verification guide, domain pages, Mermaid diagrams, onboarding notes, and an `llms.txt` proposal. Never overwrite project documentation without checking local instructions.
