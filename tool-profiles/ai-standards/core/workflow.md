# Antigravity Workflow

Use this workflow for substantive work:

1. Observe the request, repository state and applicable rules.
2. Read compact context first: global `AGENTS.md`, Orquestrador rules, Maestro protocol and project `DEV/` overview.
3. Classify the task as `fast`, `standard`, `deep`, `multiagent`, `saas` or `security`.
4. Route through `SKILL_ALIASES.json`, `SKILLS_ROUTER.json` and `SKILL_CHAINS.json`.
5. Open only the selected skill and directly referenced files.
6. Execute the smallest safe change that solves the task.
7. Run meaningful verification.
8. Update `DEV/WORKLOG.md` when project work was substantive.
9. Report what changed, what was verified and any remaining risk.

For broad work, prefer a short explicit plan before editing files.
